## Backend / production-логика

### D10 Миры — выдача ритуала

**Что изменить:** при завершении дня 10 у Миры пользователь должен получать **+1 ритуал** (как и за все предыдущие дни D1–D9).

**Текущее состояние прототипа:**
- `completeMiraDay(day)` только маркирует день как пройденный (`sw-mira-days`), счётчик ритуалов (`sw-rituals-done`) **не инкрементируется** ни на одном дне.
- В прототипе единственная точка инкремента ритуала — `completeRitualTask()` (использовалась раньше в подарке Хлои, который из флоу убран).

**Что нужно в production:**
- За каждый завершённый день Миры (D1–D10) — `+1 ритуал` в счётчик пользователя.
- На главной (Home) счётчик ритуалов растёт по 1 за день.
- К моменту знакомства с Хлоей (после D10) у пользователя в счётчике уже **10 ритуалов**, накопленных за серию.

**Почему важно:** Хлоя на онбординге обещает "за каждое задание — один ритуал". Если backend не выдаёт ритуал за дни Миры, юзер на Home видит 0/10 и ломается доверие к словам Хлои. Подарок Хлои на D10 убран именно потому, что D10 уже даёт ритуал за день — дублировать не нужно.

**Для production:**
- Убрать экран "подарок ритуала" из onboarding-флоу.

## Разблокировка Арены на карте мира (s17)

**Что добавлено в прототип:**
- На клик "По рукам" (sD10f) — ставится флаг `localStorage["sw-khloe-met"] = "1"`.
- На карте мира (s17) пин **Арена** имеет два состояния:
  - **Locked** (по умолчанию) — приглушённый, замочек, не кликабелен. ID: `#map-arena-locked`.
  - **Active** (после знакомства с Хлоей) — pill в стиле Академии (просто белая надпись "Арена", без пульса и подписи), клик ведёт на `sArena`. ID: `#map-arena-active`.
- Функция `updateMapState()` читает `sw-khloe-met` и переключает `display` между двумя нодами. Вызывается на загрузку страницы и при входе на s17.
- Ресет флага добавлен в `resetAllQuests()` — кнопка "↺ Сбросить квесты" возвращает Арену в locked.

**Что нужно в production:**
- После завершения D10 у Миры + знакомства с Хлоей (нажатие "По рукам" на sD10f) — у юзера в state открывается доступ к локации **Арена**.
- На карте мира локация Арена становится **активной**: визуально — обычный белый pill с надписью "Арена" (без пульса, без подписи — как Академия).
- Тап по Арене ведёт на экран Хлои (см. ниже **sArena**).

**Почему важно:** связка "приходи завтра" → Home → видишь арену с пульсом "Хлоя ждёт" → тап → разговор с Хлоей. Без этого после онбординга у юзера тупик — Хлоя сказала "приходи", но прийти некуда.

## Новый экран sArena (Арена: Хлоя)

**Где:** в HTML после `sD10h`, в сайдбаре "82. Арена: Хлоя".

**Содержимое:**
- Header: ← (chevron на возврат к карте `s17`) + надпись "Арена".
- Картинка Хлои `khloe-v1-emerald-bust.jpg` (height 280px, как на sD10e/sD10f).
- Бабл с тем же текстом, что на sD10f (повторение правил для юзера, который зашёл к Хлое снова):
  > Каждый день я буду давать тебе задание. За каждое — один ритуал.
  > 
  > Всё просто. Выполнил — получил. Не выполнил — ритуал сгорает.
  > 
  > Приходи завтра — буду ждать тебя на Арене. Найдёшь её на карте мира.
- **Без CTA-кнопки** — выход только через back-стрелку. Это не онбординг-узел, а локация для повторного захода.

**Зачем:** юзеру нужно место, куда можно прийти к Хлое и услышать правила ритуала ещё раз (если забыл / любопытно / тестит). На следующих итерациях этот экран превратится в "ежедневный цикл Хлои" — каждый день она будет выдавать новое задание.

**Для production:**
- Создать экран Арены с прокидыванием диалога Хлои.
- Текст диалога — копия sD10f (на этом этапе, до запуска ежедневного цикла).
- Изображение — `khloe-v1-emerald-bust.jpg`.
- Навигация back → карта мира.
